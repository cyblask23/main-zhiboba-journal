# main-zhiboba-journal

## 项目简介

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。所有页面均以静态 HTML 文件的形式存放，不依赖后端服务或特定域名。本仓库仅用于资料整理与页面存档，不针对任何特定网站或平台。

## 目录结构

```
main-zhiboba-journal/
├── README.md          # 本说明文件
├── index.html         # 主入口页面（可选）
├── pages/             # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/            # 公共资源（CSS、JS、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：包含所有独立归档的 HTML 页面文件。
- `assets/`：存放页面共享的样式、脚本及图片等资源文件。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，可直接在浏览器中打开查看。
- 页面之间不强制关联，可根据需要单独访问或引用。
- 归档内容以静态形式保存，不包含动态交互或外部依赖（除非在页面内明确引用）。

## 维护说明

- 本仓库仅用于归档和展示，不保证内容的时效性或完整性。
- 如需新增或更新页面，请将 HTML 文件放入 `pages/` 目录，并确保资源引用路径正确。
- 建议在 `assets/` 中管理公共资源，避免重复。
- 欢迎提交 Issue 或 Pull Request 以改进页面质量或修复问题。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任一 HTML 文件。
3. 也可部署到任意静态托管服务（如 GitHub Pages、Netlify 等）进行在线访问。
